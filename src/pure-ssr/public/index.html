Pure Server Side Rendering

HTML is fully rendered on the server and sent to the client. No JavaScript is needed.

Technology Stack

Frontend

HTML5
CSS

Backend

Express.js
Static File Serving

Request Flow

1

Client Request

Browser requests page from server

2

Server Response

Server sends complete HTML document

3

Browser Render

Browser displays the page immediately

See for Yourself

1

Open DevTools

Press F12 or right-click and select "Inspect"

2

Network Tab

Click the Network tab and refresh the page

3

Observe

You'll see just two files: index.html and styles.css

4

Verify

No JavaScript files are downloaded or executed

Advantages

Fast Initial Load

No JavaScript to parse or execute, leading to quick page loads

SEO Friendly

Content is immediately available for search engines to crawl

Low Complexity

Simple architecture with minimal points of failure

Disadvantages

Limited Interactivity

No dynamic updates or complex user interactions without page reloads

Full Page Reloads

Every navigation requires a complete page refresh from the server

No State Management

Cannot maintain client-side state between page transitions
